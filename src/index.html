Inline Elements

Characteristics

  • Flows with the line (text), soft wraps within container
  • Width is determined by contents
  • Height is determined by contents
  • Setting Width or Height has no impact

Examples

<span>, <a>, <em>, <strong>, <code>, <abbr>, <small>, <i>, <b>

Characteristics

  • Width is determined by container
  • Height is determined by contents

Examples

<div>, <h1-h6>, <p>, <ul>, <ol>, <li>, <section>, <article>, <aside>, <main>, <header>, <footer>, <table> (quirky), <fieldgroup>

Choosing the Right Element

If It Goes With the Flow

  • Use <a>, <video>, <img>, <input>, and etc. when those functions or media types are reqiured
  • Use <span> for pretty much everything else
  • <abbr>, <i>, <b>, <em>, <strong>, and etc. have well defined semantics for web pages but are a bit controversial for web applications. We need to have clear guidelines when choosing these elements.

If It Looks Like a Block and Acts Like a Block

  • Use <header>,<footer>, <main>, <div class="main"> for page structure
  • Use <p> for paragraphs
  • Use <h1>-<h6> for headings
  • Use <div> for pretty much everything else
  • <section>, <aside>, and <article> have well defined semantics for web pages but are a bit controversial for web applications. We need to have clear guidelines when choosing these elements.

Choose the element that is behaving closest to the design to avoid excessive styling You will be glad you did

Don't put block elements inside inline elements You're doing it wrong

Positioning Elements

Floating Elements

Characteristics

  • Removes the element from the natural flow. Inline elements and text will flow around it
  • Width is determined by contents
  • Height is determined by contents
  • Width and Height can be explictly set
  • Floated element does not influence the size of its container *

Floating Elements

Pro-tips

  • For best cross-browser compatability, when using float: left|right always set display:block
  • For accessiliblity don't float elements unless they visually display in the same order as the HTML markup
  • * Extra CSS can be used to "enclose" a floated element

Relative Positioned Elements

Characteristics

  • Shifts the element relatively from its normal flow but doesn't alter the line
  • Inline width is determined by contents
  • Block width is determined by container
  • Height is determined by contents
  • Width and Height can be explictly set
  • Relatively positioned element influence the size of its container
  • Affects the positioning of its contents

Absolute Positioned Elements

Characteristics

  • Moves the element out of the line but doesn't affect the flow of other elements
  • Width is determined by contents
  • Height is determined by contents
  • Width and Height can be explictly set
  • Rendered Top/Left (or Bottom/Right) of its first relatively positioned ancestor or body
  • Does not influence the size of its container *

Absolute Positioned Elements

Pro-tips

  • Absolutely positioned element acts as a relative container to its content that is absolutlely positioned
  • Setting Top and Bottom ignores Bottom
  • Setting Left and Right ignores Right

Fixed Positioned Elements

Characteristics

  • Moves the element out of the line but doesn't affect the flow of other elements
  • Width is determined by contents
  • Height is determined by contents
  • Width and Height can be explictly set
  • Rendered Top/Left (or Bottom/Right) of the viewport
  • Does not move with scrolling of container
  • Does not influence the size of its container *
